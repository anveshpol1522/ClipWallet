# The Soul of Open Source

Open source is not a numbers game or a resume checkbox.

This project exists to encourage:
- real collaboration
- thoughtful engineering
- maintainable contributions
- respectful communication
- solving genuine problems

A single high-quality contribution is valued far more than multiple superficial pull requests.

Please respect maintainers' time:
- Read documentation before asking questions
- Verify existing issues before creating duplicates
- Follow contribution instructions carefully
- Communicate clearly and honestly

---

# Before You Contribute

## 1. Check Existing Issues

Before opening a new issue:
- Verify the issue does not already exist
- Search through open and closed issues
- Avoid duplicates

Duplicate or spam issues may be closed immediately.

---

## 2. Create an Issue Before Opening a PR

You **must create an issue before submitting a pull request**.

Rules:
- Explain the problem clearly
- Wait for maintainer verification
- Wait for approval before starting work
- Do not submit unsolicited pull requests

Pull requests without approved issues may be closed without review.

---

## 3. Maintainer-Assigned Tasks

Maintainers will regularly publish:
- beginner tasks
- feature requests
- bug fixes
- optimization tasks
- documentation improvements

Contributors are encouraged to work on officially listed issues first.

# AI Usage Policy

AI usage is allowed in this repository **with strict conditions**. Developer must have clear understanding about the code they are generating and the architecture they are working on. Genereating PRs without understanding the codebase is not allowed and may be considered as violation of code of conduct. Spammy AI generated PRs may be closed without review. 

## Allowed AI Usage

You may use AI tools such as:
- GitHub Copilot
- ChatGPT
- Claude
- Cursor
- Other AI assistants

Examples of acceptable usage:
- debugging help
- understanding concepts
- generating boilerplate
- improving architecture ideas
- productivity assistance

---
